# Oh My Skills v0.1.1

## 中文

### 概述

v0.1.1 是 v0.1.0 之后的首个小版本更新，重点补齐「中心库」独立工作区、同名 Skill 识别，以及一轮界面与跨平台体验优化。

### 下载

| 平台 | 文件 |
|------|------|
| macOS（Apple Silicon） | `Oh My Skills_0.1.1_aarch64.dmg` |

安装：打开 DMG，将应用拖入「应用程序」文件夹。  
若首次打开被系统拦截，请到「系统设置 → 隐私与安全性」允许，或右键应用选择「打开」。

> 说明：当前为本地构建版本，尚未做 Apple 公证与自动更新。Windows 安装包仍沿用 v0.1.0 发布流程，本次主要发布 macOS DMG。

### 新功能

**中心库工作区**  
Skills 页新增第三个工作区：**中心库**。全局、项目、中心库分开查看，不再混在一起：

- **全局**：各 Agent 机器级生效的 Skills
- **项目**：按项目根目录查看项目级 Skills
- **中心库**：已导入、可统一分发的规范副本

从全局或项目导入 Skill 后，可在中心库集中管理，再同步到其他 Agent。

**同名 Skill 识别更准确**  
当多个路径存在相同 slug、但内容不同的 Skill 时，应用会按「slug + 身份」区分，避免列表里误合并成一条。

### 改进

- **诊断提示更清晰**：`name-mismatch`（目录名与 SKILL.md frontmatter 名称不一致）的说明更易理解
- **同步结果展示**：执行同步后的结果区域布局与可读性更好
- **列表多选体验**：选中 Skill 后，底部选择栏与列表滚动区域不再互相遮挡
- **界面视觉统一**：窗口背景与内容区配色调整，macOS / Windows 窗口透明度设置更一致
- **跨平台构建**：补充 macOS / Windows 分平台构建命令与配置，完善 CI 发布流程
- **文档**：README 增加产品截图；补充 `AGENTS.md` 协作说明

### 修复

- 修正同名 Skill 在列表中被错误聚合的问题
- 优化名称不一致类问题的提示文案

### 已知限制

- 尚未提供代码签名、公证与自动更新
- Windows 软链接同步仍依赖 Developer Mode 或管理员权限；不可用时会回退为复制
- 仍为 MVP 阶段，功能以「发现 → 预览 → 确认 → 同步」为主流程

### 从 v0.1.0 升级

直接安装 v0.1.1 覆盖即可，中心库路径与已有 Skills 数据会保留。

## English

### Overview

v0.1.1 is the first patch release after v0.1.0. It adds a dedicated **Central Library** workspace, improves duplicate-skill identity handling, and polishes the UI and cross-platform build experience.

### Downloads

| Platform | File |
|----------|------|
| macOS (Apple Silicon) | `Oh My Skills_0.1.1_aarch64.dmg` |

Install by opening the DMG and dragging the app into Applications.  
If macOS blocks the app on first launch, allow it under **System Settings → Privacy & Security**, or right-click the app and choose **Open**.

> Note: This is a local release build without Apple notarization or auto-update. The Windows installer still follows the v0.1.0 release flow; this release primarily ships the macOS DMG.

### What's New

**Central Library workspace**  
The Skills page now has a third workspace tab: **Library**. Global, project, and library views are separated:

- **Global**: machine-level skills active across agents
- **Project**: project-scoped skills grouped by project root
- **Library**: normalized copies imported into your central library for redistribution

After adopting skills from global or project scopes, manage them centrally in the library and sync them to other agents.

**More accurate duplicate-skill handling**  
When multiple paths share the same slug but different content, skills are grouped by **slug + identity** so the list no longer collapses them incorrectly.

### Improvements

- Clearer diagnostics for `name-mismatch` (directory name vs. SKILL.md frontmatter name)
- Better layout and readability for sync execution results
- Improved multi-select bar behavior so the bottom action bar no longer overlaps the skill list
- Unified window and content background styling across macOS and Windows
- Platform-specific build commands and CI release workflow updates
- README screenshots and contributor guidance in `AGENTS.md`

### Fixes

- Fixed incorrect aggregation of skills that share a slug but differ in content
- Improved messaging for name-mismatch issues

### Known Limitations

- No code signing, notarization, or auto-update yet
- Windows symlink sync still requires Developer Mode or administrator privileges; copy fallback remains available
- Still an MVP focused on discover → preview → confirm → sync

### Upgrading from v0.1.0

Install v0.1.1 over the existing app. Your library path and existing skill data are preserved.
